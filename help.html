Security Operations Center (SOC) Simulation Tutorial

Welcome to the SOC Analyst Simulation. In this educational software, you will take on the role of a Tier 1 SOC Analyst responsible for monitoring, triaging, and responding to security alerts. Your primary objective is to evaluate incoming alerts and correctly classify them based on the evidence provided.

The Analyst Dashboard

The main interface consists of an alert panel displaying realistic security events generated by the network. Each alert contains metadata that you must analyze, which may include:

  • Timestamp of the event
  • Source and destination IP addresses
  • Event descriptions or specific log messages
  • Usernames, file paths, or process names involved

Classification Categories

For every alert you review, you must investigate the details and make a determination by clicking one of the three action buttons:

  1. False Alarm (False Positive): Select this if the alert was triggered by legitimate, authorized activity or a misconfiguration rather than malicious behavior. Examples include a known administrative tool being used by an authorized user or routine scheduled backups.
  2. Potential Alarm (Benign Positive / Suspicious): Select this if the activity is unusual or suspicious but does not show definitive proof of a high-risk compromise. These require further monitoring or context, such as a user logging in from an uncommon location or a single failed login attempt.
  3. Critical Alarm (True Positive / Incident): Select this if the alert shows clear evidence of unauthorized access, active exploitation, malware execution, or data exfiltration. Examples include confirmed ransomware deployment, successful brute-force attacks, or known malicious command-and-control communication.

Workflow and Investigation Tips

  • Read the Details: Examine the log data carefully. Look for anomalies in port numbers, unusual command-line arguments, or unexpected timestamps.
  • Correlate Data: Compare the source and destination information. Is an internal server communicating with an unknown external IP address?
  • Make Your Decision: Once you have assessed the risk level, click the corresponding button to log your decision and move to the next alert. Your performance will be evaluated based on the accuracy of these classifications.
